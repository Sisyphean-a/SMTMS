# 模组列表打开文件夹设计

**目标**：在主列表界面的名称列左侧增加一个 `Open` 按钮列，点击后打开对应模组所在文件夹。

**背景**：主列表由 `src/SMTMS.Avalonia/Views/ModListView.axaml` 的 `DataGrid` 渲染，每行数据为 `ModViewModel`。每个模组都已有 `ManifestPath`，可据此解析出所在目录。

## 方案比较

### 方案 1：新增独立 `Open` 列（采用）
- 在名称列左侧新增一个窄的 `DataGridTemplateColumn`
- 每行显示文字按钮 `Open`
- 按钮通过父级 `ModListViewModel` 命令处理点击

**优点**
- 与需求完全一致
- 不污染现有名称列展示逻辑
- 改动范围小，风险低

**缺点**
- 列表横向空间略有增加

### 方案 2：把名称列改成复合模板
- 在名称列内部同时放 `Open` 按钮和名称文本

**优点**
- 不增加总列数

**缺点**
- 名称列职责混杂
- 与“左侧增加一列”的需求不一致

### 方案 3：右键菜单打开文件夹
- 通过上下文菜单提供打开动作

**优点**
- 界面更简洁

**缺点**
- 发现性差
- 不符合按钮列需求

## 最终设计

### UI
- 在 `ModListView.axaml` 的名称列前新增 `Open` 列
- 列头显示 `Open`
- 单元格使用文字按钮 `Open`
- 按钮提示为“打开模组文件夹”

### 交互
- 点击按钮后，读取当前行 `ModViewModel.ManifestPath`
- 取其目录作为目标文件夹
- 调用平台服务打开该目录

### 架构
- 不直接在 `ModViewModel` 中继续堆叠外部进程调用逻辑
- 在 `ModListViewModel` 中新增命令，接收当前行作为参数
- 新增一个小型服务接口，封装 `UseShellExecute=true` 的路径打开行为，便于测试

### 错误处理
- `ManifestPath` 为空：显示错误状态消息
- 目录解析失败：显示错误状态消息
- 外部打开失败：记录日志并显示错误状态消息

### 测试
- 为 `ModListViewModel` 增加回归测试
- 验证传入合法 `ManifestPath` 时，会把目录路径传给打开服务
- 验证路径无效时，不调用打开服务

### 约束
- 只打开模组目录，不打开 `manifest.json` 文件本身
- 保持现有列表排序、选择、详情面板行为不变
